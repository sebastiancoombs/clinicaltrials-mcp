ClinicalTrials.gov MCP x402 USDC on Base

Pay-per-call MCP for ClinicalTrials.gov v2. Three endpoints, no signup, no API key. Settle USDC on Base via x402.

Endpoints

EndpointSourcePrice
GET /v1/trials/search ClinicalTrials.gov v2 study search $0.05
GET /v1/trials/by_nct ClinicalTrials.gov v2 single-study record $0.05
GET /v1/sponsors/pipeline ClinicalTrials.gov v2 sponsor pipeline aggregation $0.07

Discovery surfaces

  • /.well-known/agent-card.json — Google A2A AgentCard
  • /.well-known/mcp.json — MCP server manifest
  • /.well-known/ai-plugin.json — OpenAI plugin manifest
  • /openapi.yaml — OpenAPI 3.1 spec
  • /mcp — MCP Streamable HTTP transport (JSON-RPC 2.0)

Quick start (curl)

# 1) First call returns 402 with x402 envelope:
curl -i 'https://clinicaltrials-mcp.mtree.workers.dev/v1/trials/search?q=cancer&phase=PHASE3&status=RECRUITING&size=5'

# 2) Settle the envelope (sign EXACT-scheme USDC payment on Base, payTo as advertised),
#    then retry with X-PAYMENT header. The response is the actual data.

# Other examples:
curl -i 'https://clinicaltrials-mcp.mtree.workers.dev/v1/trials/by_nct?nct=NCT04368728'
curl -i 'https://clinicaltrials-mcp.mtree.workers.dev/v1/sponsors/pipeline?sponsor=Pfizer&size=25'

Why this exists

ClinicalTrials.gov is fully open at the source, but the agent-friendly access pattern isn't: nested JSON Schema-of-Schemas, multi-page Atom-style flow control, and you have to parse the protocolSection/identificationModule/identificationModule.nctId tree yourself. This service is the x402-native door — agents pay USDC for flat agent-callable rows, no human onboarding, no contracts, no minimum.
